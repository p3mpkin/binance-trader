# Binance Trader 中文使用文档

本文档整理当前仓库里最常用的 3 个命令：

- `scan_movers.py`：扫描异动标的，只输出候选，不开仓。
- `trader_bollinger.py`：针对单个标的运行布林带策略。
- `auto_trade_movers.py`：自动扫描异动标的，策略二次确认后跟踪并开/平仓。

默认建议先用纸交易模式运行。只有在充分验证后，再考虑实盘。

## 运行前准备

激活 conda 环境：

```bash
cd /Users/p3mpkin/Desktop/code/binance-trader
conda activate binance-trader
```

如果使用实盘，需要配置：

```bash
app/config.py
```

填入：

```python
api_key = '你的 Binance API Key'
api_secret = '你的 Binance API Secret'
```

API 权限建议只开需要的交易权限，不要开启提现权限。

## 推荐运行顺序

1. 扫描异动标的：

```bash
python scan_movers.py --market_type futures --interval 5m --top 20
```

2. 挑一个标的单独纸交易：

```bash
python trader_bollinger.py \
  --symbol BTCUSDT \
  --position_pct 5 \
  --paper_balance 1000 \
  --market_type futures \
  --leverage 20 \
  --futures_side BOTH \
  --move_exits \
  --test_mode
```

3. 启动自动扫描 + 自动跟踪纸交易：

```bash
python -u auto_trade_movers.py \
  --strategy_mode mean_reversion \
  --position_pct 5 \
  --paper_balance 1000 \
  --leverage 20 \
  --futures_side BOTH \
  --move_exits \
  --max_positions 3 \
  --entries_per_scan 1
```

默认会把纸交易状态保存到 `paper_state.json`。如果程序因为网络波动、终端关闭等原因中断，下一次可以加 `--resume_state` 继续恢复模拟仓位：

```bash
python -u auto_trade_movers.py \
  --strategy_mode mean_reversion \
  --position_pct 5 \
  --paper_balance 1000 \
  --leverage 20 \
  --futures_side BOTH \
  --move_exits \
  --max_positions 3 \
  --entries_per_scan 1 \
  --state_file paper_state.json \
  --resume_state
```

`-u` 表示实时输出日志，推荐用于长期运行脚本。

## 1. scan_movers.py 异动扫描

用途：扫描 Binance 标的，找出成交量、波动率、布林带位置异常的候选。它不会开仓。

常用命令：

```bash
python scan_movers.py \
  --market_type futures \
  --interval 5m \
  --min_quote_volume 50000000 \
  --top 20
```

更激进，扫描更多标的：

```bash
python scan_movers.py \
  --market_type futures \
  --interval 5m \
  --min_quote_volume 0 \
  --candidates 500 \
  --top 50
```

网络不稳定时：

```bash
python scan_movers.py \
  --market_type futures \
  --interval 5m \
  --workers 3 \
  --retries 3 \
  --retry_wait 1
```

### scan_movers.py 参数

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `--market_type` | `futures` | 扫描市场，支持 `futures` 或 `spot`。 |
| `--quote_asset` | `USDT` | 只扫描指定计价资产，例如 `USDT`。 |
| `--interval` | `5m` | 用于异动判断的 K 线周期。 |
| `--scan_mode` | `trend` | 扫描方向模式，`trend` 顺势突破，`mean_reversion` 反向回归。 |
| `--kline_limit` | `60` | 每个标的拉取多少根 K 线。 |
| `--candidates` | `120` | 按 24h 成交额排序后，最多深入扫描多少个标的。 |
| `--top` | `20` | 最终输出前多少个结果。 |
| `--min_quote_volume` | `50000000` | 24h 成交额过滤，小于该值不扫描。 |
| `--min_score` | `0` | 最低异动综合分。 |
| `--min_volume_ratio` | `0` | 最近一根 K 线成交量 / 过去均量的最低倍数。 |
| `--min_range_pct` | `0` | 最近一根 K 线最低振幅百分比。 |
| `--bb_period` | `20` | 布林带周期。 |
| `--bb_stddev` | `2.0` | 布林带标准差倍数。 |
| `--workers` | `8` | 并发请求数量。网络不稳时建议调低。 |
| `--retries` | `2` | 单个标的 K 线请求失败后的重试次数。 |
| `--retry_wait` | `0.5` | 每次重试之间等待秒数。 |

### scan_movers.py 输出字段

| 字段 | 说明 |
| --- | --- |
| `Symbol` | 标的，例如 `BTCUSDT`。 |
| `Score` | 异动综合分，越高越异常。 |
| `Dir` | 观察方向，`LONG`、`SHORT`、`WATCH`。不是下单指令。 |
| `24h%` | 24 小时涨跌幅。 |
| `Lookbk%` | 本次 K 线窗口内涨跌幅。 |
| `VolX` | 最近成交量相对过去均量的倍数。 |
| `Range%` | 最近一根 K 线振幅。 |
| `BBW%` | 布林带宽度。 |
| `%B` | 当前价格在布林带里的位置。小于 0 表示低于下轨，大于 1 表示高于上轨。 |
| `RSI` | RSI 数值。 |
| `QuoteVol` | 24h 成交额。 |
| `Signal` | 扫描信号，例如 `breakout_up`、`breakdown`、`near_upper`、`near_lower`、`inside`。 |

## 2. trader_bollinger.py 单标的交易

用途：对单个 symbol 运行布林带策略。支持纸交易、现货、USD-M 合约。

纸交易示例：

```bash
python trader_bollinger.py \
  --symbol BTCUSDT \
  --position_pct 5 \
  --paper_balance 1000 \
  --market_type futures \
  --leverage 20 \
  --futures_side BOTH \
  --move_exits \
  --test_mode
```

实盘示例：

```bash
python trader_bollinger.py \
  --symbol BTCUSDT \
  --position_pct 5 \
  --market_type futures \
  --leverage 20 \
  --futures_side BOTH \
  --move_exits
```

实盘会要求输入 `START` 确认。

### trader_bollinger.py 参数

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `--symbol` | 必填 | 交易对，例如 `BTCUSDT`。 |
| `--quantity` | `0` | 固定交易数量。大于 0 时优先使用。 |
| `--amount` | `0` | 固定名义金额，例如 `100` 表示约 100 USDT。 |
| `--position_pct` | `0` | 使用余额/权益的百分比作为保证金，例如 `5` 表示 5%。优先级高于 `--amount`。 |
| `--paper_balance` | `1000` | 纸交易时用于百分比仓位计算的虚拟余额。 |
| `--market_type` | `futures` | 市场类型，`futures` 或 `spot`。 |
| `--leverage` | `20` | USD-M 合约杠杆。实盘会向 Binance 设置杠杆。 |
| `--futures_side` | `LONG` | 合约方向，`LONG`、`SHORT`、`BOTH`。 |
| `--strategy_mode` | `mean_reversion` | 策略模式，`mean_reversion` 均值回归，`breakout` 趋势突破。 |
| `--bb_period` | `20` | 布林带周期。 |
| `--bb_stddev` | `2.0` | 布林带标准差倍数。 |
| `--rsi_period` | `14` | RSI 周期。 |
| `--rsi_oversold` | `30` | RSI 超卖阈值。 |
| `--rsi_overbought` | `70` | RSI 超买阈值。 |
| `--volume_threshold` | `1.2` | 成交量确认阈值。 |
| `--stop_loss_atr` | `2.0` | 止损距离，按 ATR 倍数计算。 |
| `--take_profit` | `middle` | 旧版止盈目标，支持 `middle`、`upper`、数字百分比或 `trailing`。 |
| `--take_profit_strategy` | `legacy` | 止盈策略，支持 `legacy`、`band`、`percent`、`atr`、`risk_reward`、`trailing`。 |
| `--take_profit_value` | `2.0` | `percent`、`atr`、`risk_reward` 模式使用的数值。 |
| `--take_profit_band` | `middle` | `band` 模式目标，`middle` 为中轨，`outer` 为外轨。 |
| `--risk_per_trade` | `2.0` | 动态仓位模式下的风险百分比。 |
| `--move_exits` | 关闭 | 持仓期间根据最新 ATR 和布林带移动止损止盈。 |
| `--min_bb_width` | `1.0` | 最小布林带宽度过滤。低于该值不交易。 |
| `--max_bb_width` | `10.0` | 最大布林带宽度过滤。高于该值不交易。 |
| `--min_confidence` | `50` | 策略信号最低置信度。 |
| `--interval` | `5m` | 策略使用的 K 线周期。 |
| `--kline_limit` | `100` | 每轮拉取的历史 K 线数量。 |
| `--wait_time` | `10` | 每轮分析之间等待秒数。 |
| `--max_trades` | `0` | 最大完成交易数，`0` 表示不限。 |
| `--test_mode` | 关闭 | 纸交易模式，虚拟开平仓，不真实下单。 |
| `--debug` | 关闭 | 输出详细日志。 |
| `--commission` | `BNB` | 手续费估算类型，`BNB` 或 `TOKEN`。 |

### 仓位计算规则

优先级：

1. `--quantity` 大于 0：使用固定数量。
2. `--position_pct` 大于 0：使用余额百分比。
3. `--amount` 大于 0：使用固定名义金额。
4. 否则使用策略动态仓位。

合约百分比仓位计算：

```text
保证金 = 余额 * position_pct / 100
名义仓位 = 保证金 * leverage
数量 = 名义仓位 / 入场价
```

例如：

```text
paper_balance = 1000
position_pct = 5
leverage = 20
保证金 = 1000 * 5% = 50 USDT
名义仓位 = 50 * 20 = 1000 USDT
```

## 3. auto_trade_movers.py 自动扫描交易

用途：自动扫描异动标的，使用策略二次确认后开仓，并持续跟踪仓位。

默认是纸交易，不会真实下单。

推荐纸交易命令：

```bash
python -u auto_trade_movers.py \
  --strategy_mode mean_reversion \
  --position_pct 5 \
  --paper_balance 1000 \
  --leverage 20 \
  --futures_side BOTH \
  --move_exits \
  --max_positions 3 \
  --entries_per_scan 1
```

实盘命令：

```bash
python -u auto_trade_movers.py \
  --strategy_mode mean_reversion \
  --position_pct 5 \
  --leverage 20 \
  --futures_side BOTH \
  --move_exits \
  --max_positions 3 \
  --entries_per_scan 1 \
  --live
```

实盘会要求输入 `START`。

### auto_trade_movers.py 工作流程

```text
扫描异动标的
-> 取 Top 候选
-> 根据 Dir 判断候选方向
-> 对候选再跑 BollingerStrategy 二次确认
-> LONG 候选必须确认 Signal=BUY
-> SHORT 候选必须确认 Signal=SELL
-> 开仓
-> 持续跟踪持仓
-> 移动止盈止损
-> 触发止损/止盈/反向信号后平仓
```

它不会只根据扫描器 `Dir` 直接开仓，会先做策略确认。

| 参数 | 默认值 | 说明 |

| `--quote_asset` | `USDT` | 扫描计价资产。 |
| `--scan_interval` | `5m` | 扫描和交易确认使用的 K 线周期。 |
| `--scan_mode` | `auto` | 扫描方向模式，`auto` 会跟随策略模式：`breakout` 用 `trend`，`mean_reversion` 用 `mean_reversion`。 |
| `--scan_every` | `60` | 每隔多少秒重新扫描一次。 |
| `--candidates` | `120` | 每轮最多深入扫描多少个高流动性标的。 |
| `--top` | `10` | 每轮最多考虑多少个扫描结果。 |
| `--min_quote_volume` | `50000000` | 24h 成交额过滤。 |
| `--min_mover_score` | `60` | 扫描异动分至少达到该值才进入策略确认。 |

## 策略信号说明

当前支持两种策略模式。

### mean_reversion 均值回归

```text
价格接近/跌破下轨 => BUY
价格接近/突破上轨 => SELL
```

适合震荡行情，思路是“极端位置回归中轨”。

### breakout 趋势突破

```text
价格突破上轨 + 放量 + 动能确认 => BUY
价格跌破下轨 + 放量 + 动能确认 => SELL
```

适合趋势行情，思路是“突破后顺势跟随”。

趋势模式推荐命令：

```bash
python -u auto_trade_movers.py \
  --strategy_mode breakout \
  --position_pct 5 \
  --paper_balance 1000 \
  --leverage 20 \
  --futures_side BOTH \
  --move_exits \
  --min_mover_score 60 \
  --min_confidence 60
```

交易策略返回 4 种信号：

| 信号 | 说明 |
| --- | --- |
| `BUY` | 价格接近布林带下轨，同时 RSI、成交量、MACD 条件加分足够。合约中用于开多或平空。 |
| `SELL` | 价格接近布林带上轨，同时 RSI、成交量、MACD 条件加分足够。合约中用于开空或平多。 |
| `TAKE_PROFIT` | 价格回到布林带中轨附近。主要用于多单止盈。 |
| `WAIT` | 没有满足交易条件，不操作。 |

只有当：

```text
Signal 符合方向
Confidence >= min_confidence
```

才会开仓或平仓。

## 止盈策略说明

当前支持 6 种止盈策略：

| 策略 | 参数 | 说明 |
| --- | --- | --- |
| `legacy` | `--take_profit middle/upper/数字/trailing` | 兼容旧参数。`middle` 表示中轨，`upper` 表示多单上轨/空单下轨，数字表示百分比。 |
| `band` | `--take_profit_band middle/outer` | 按布林带止盈。`middle` 到中轨止盈，`outer` 到外轨止盈。 |
| `percent` | `--take_profit_value 3` | 固定百分比止盈。例如多单 +3%，空单 -3%。 |
| `atr` | `--take_profit_value 4` | ATR 倍数止盈。例如多单入场价 + 4 * ATR。 |
| `risk_reward` | `--take_profit_value 2` | 按风险收益比止盈。例如 2R。 |
| `trailing` | 无固定止盈 | 不设固定止盈价，只依赖移动止损和反向信号退出。 |

示例：固定 3% 止盈：

```bash
python -u auto_trade_movers.py \
  --strategy_mode breakout \
  --take_profit_strategy percent \
  --take_profit_value 3 \
  --position_pct 5 \
  --leverage 20 \
  --futures_side BOTH \
  --move_exits
```

示例：2R 止盈：

```bash
python -u auto_trade_movers.py \
  --strategy_mode breakout \
  --take_profit_strategy risk_reward \
  --take_profit_value 2 \
  --position_pct 5 \
  --leverage 20 \
  --futures_side BOTH
```

示例：只用移动止损：

```bash
python -u auto_trade_movers.py \
  --strategy_mode breakout \
  --take_profit_strategy trailing \
  --position_pct 5 \
  --leverage 20 \
  --futures_side BOTH \
  --move_exits
```

## 移动止盈止损说明

开启：

```bash
--move_exits
```

多单：

```text
止损 = 当前价 - ATR * stop_loss_atr
止盈跟随布林带上轨
只往有利方向移动
```

空单：

```text
止损 = 当前价 + ATR * stop_loss_atr
止盈跟随布林带下轨
只往有利方向移动
```

## 纸交易状态恢复

`auto_trade_movers.py` 默认是模拟交易，仓位原本只存在内存里。现在脚本会把纸交易状态写入本地 JSON 文件：

```bash
paper_state.json
```

保存内容包括：

```text
未平仓模拟仓位
累计已实现收益
手续费估算
胜负次数
同一标的冷却时间
```

恢复命令：

```bash
python -u auto_trade_movers.py \
  --position_pct 5 \
  --paper_balance 1000 \
  --leverage 20 \
  --futures_side BOTH \
  --move_exits \
  --state_file paper_state.json \
  --resume_state
```

注意：

- 不加 `--resume_state` 时，会从空仓开始模拟。
- `--state_file` 默认是 `paper_state.json`，也可以改成其他文件名。
- 这个功能只恢复模拟盘状态；真实实盘仓位仍然以币安账户为准。
- 如果程序中断期间价格已经触发过止盈/止损，恢复后不会回放中断期间的每根 K 线，只会从恢复后的实时价格继续判断。

## 安全建议

- 先用 `--test_mode` 或默认纸交易跑足够长时间。
- 自动交易脚本默认纸交易，只有加 `--live` 才实盘。
- 合约 20x 风险很高，`position_pct=5` 表示 5% 保证金，20x 后名义仓位约等于账户权益 100%。
- API Key 不要开启提现权限。
- 网络不稳定时降低扫描并发：`--scan_workers 3`。
- 不要只看扫描器 `Dir` 开仓，自动脚本已经做了策略二次确认。
